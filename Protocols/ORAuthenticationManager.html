ORAuthenticationManager Protocol Reference

Conforms to NSObject
Declared in ORAuthenticationManager.h

Overview

An authentication manager is responsible for providing credentials to be used when communicating with an OR controller that does require some form of authentication.

It is also responsible to validate server side certificates sent by OR controller.

– credential required method

This methods gets called when, during communication with an OR controller, some authentication is required.

- (NSObject<ORCredential> *)credential

Return Value

ORCredential credential to use for communication

Discussion

This methods gets called when, during communication with an OR controller, some authentication is required.

The authentication manager must in return provide credential that will be used to retry the call. If the provided credential does not provide access, this method will be called again. Returning nil indicates that the communication must proceed without credential, potentially leading to a communication error caused by un-authorized access.

It is guaranteed that this method will be called on a thread other than the main thread, potentially allowing the authentication manager to block while retrieving the credential (e.g. if asking to the user).

– acceptServer: required method

- (BOOL)acceptServer:(NSURLProtectionSpace *)protectionSpace

Parameters

NSURLProtectionSpace

The protection space that generates an authentication challenge.

Return Value

YES if connection to controller should proceed, NO otherwise

Discussion

This methods gets called when, during communication with an OR controller, a server side certificate is used and needs to be trusted.

The authentication manager must decide whether or not to proceed with the connection, based on provided information.

It is guaranteed that this method will be called on a thread other than the main thread, potentially allowing the authentication manager to block while deciding (e.g. if asking to the user).
